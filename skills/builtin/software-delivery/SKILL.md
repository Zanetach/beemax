# Software delivery

Use the `implement` route for implementation, debugging, refactoring, and repair.

Treat the user's requested outcome—not a code draft—as the commitment. Keep mutations in the parent Agent and inside the configured Profile workspace. Preserve existing user changes and never weaken secrets, approval, deployment, external-publication, or destructive-action boundaries.

Ask only when a missing product decision would materially change the outcome or authorize an irreversible/external effect. Make visible, reversible defaults for ordinary framework, naming, layout, and implementation details.

Do not claim completion from file existence or plausible code. Completion requires observed execution evidence against the acceptance criteria.
