# Use one Profile semantic Memory authority

Thruvera stores evidence, Claims, Episodes, learning receipts, contextual assessments, and projection lineage through one Profile-bound SQLite semantic authority. Search indices and projections are rebuildable views, while signed Skill files are immutable procedural artifacts referenced by that authority. We rejected a separate learning database, vector-store authority, or hidden Agent memory because competing authorities make correction, forgetting, scope enforcement, and recovery non-atomic; the accepted cost is a richer normalized schema behind one deep `MemoryLearningKernel` Module.
