# Settle learning only from correlated independent Verification

BeeMax reinforces or downgrades a component version only when an independently verified criterion can be correlated to immutable Context, Capability, Tool, Skill, Artifact, and source receipts. Unavailable Verification, cancellation, authorization denial, provider absence, or ambiguous cause yields `unknown` and zero failure weight. We rejected self-reported success and broad blame across an execution because they create false promotion and false downgrade; the cost is durable Contribution Receipts, criterion-level settlement, and conservative attribution.
