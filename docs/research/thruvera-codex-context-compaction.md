# Thruvera and Codex context compaction

Research date: 2026-07-13. Thruvera findings refer to the current workspace. Codex findings refer to local CLI `0.144.1` plus the OpenAI-owned upstream source linked below.

## Thruvera

- Thruvera composes each turn from the current request, Situation/Work Context, scoped Memory recall, verified runtime facts, active Task preservation, and progressively activated capabilities. Turn enrichment has a default 12,000-character budget and releases lower-priority whole items when they do not fit (`packages/core/src/conversation-context.ts`).
- Session history and compaction are supplied by the vendored Pi coding-agent runtime. Thruvera now derives reserve and retained-recent budgets from the selected model's actual context window, with optional Profile overrides, and applies them as runtime-only Pi settings (`packages/core/src/context-compaction.ts`, `packages/core/src/runtime.ts`, `pi/packages/coding-agent/src/core/settings-manager.ts`).
- Pi summarizes older history into a structured checkpoint containing Goal, Constraints, Progress, Decisions, Next Steps, Critical Context, and file operations. The latest compaction entry plus retained recent entries becomes model context; the append-only session still retains prior entries for reconstruction (`pi/packages/coding-agent/src/core/compaction/compaction.ts`, `pi/packages/coding-agent/src/core/session-manager.ts`).
- Thruvera adds a `session_before_compact` hook. It injects a redacted Task Preservation Envelope containing authoritative task identity, Situation, checkpoint, route, verification state, candidate result, evidence, and unresolved errors. If that envelope cannot be assembled safely, automatic compaction is cancelled instead of silently discarding durable recovery state. After compaction, Core verifies that every durable Task identity survived and persists a non-triggering authoritative recovery context into Pi's session transcript when the lossy summary omitted one (`packages/core/src/runtime.ts`, `packages/core/src/context-compaction.ts`, `packages/core/src/agent-runtime.ts`).
- Users can invoke `/compact`; Core only allows it for an idle session. Pi also compacts on threshold and on context overflow, with a bounded retry after overflow. Oversized serialized history is bounded against the compaction model's input window while recent context and Thruvera preservation instructions are retained (`apps/cli/src/profile-control.ts`, `pi/packages/coding-agent/src/core/agent-session.ts`, `pi/packages/coding-agent/src/core/compaction/compaction.ts`).
- `/usage` exposes the effective trigger/reserve/retained budgets, and Gateway JSONL records content-free started/completed/failed events plus preservation recovery counts (`packages/core/src/agent-runtime.ts`, `apps/cli/src/profile-control.ts`, `apps/cli/src/gateway.ts`).
- Common Pi file and shell tools retain their native line/byte limits. Thruvera additionally applies one Profile-scoped estimated-token ceiling at Pi's `tool_result` seam, so built-ins, first-class Thruvera Tools, and MCP adapters share the same context-protection budget (`packages/core/src/tool-runtime.ts`, `packages/core/src/runtime.ts`, `pi/packages/coding-agent/src/core/extensions/types.ts`).

## Codex

- Codex supports automatic and manual thread compaction. Its threshold is model-specific and can be overridden through `model_auto_compact_token_limit`; current configuration also supports choosing whether the limit applies to full context or tokens after the carried prefix. [OpenAI configuration schema](https://github.com/openai/codex/blob/main/codex-rs/core/config.schema.json)
- Current Codex can use provider-side remote compaction or inline model summarization. Its summary prompt asks for progress, decisions, constraints, remaining work, and critical references. [OpenAI compact implementation](https://github.com/openai/codex/blob/main/codex-rs/core/src/compact.rs) and [official compact prompt](https://github.com/openai/codex/blob/main/codex-rs/core/templates/compact/prompt.md)
- The replacement history retains a bounded tail of real user messages (20,000 approximate tokens), adds the generated handoff summary, and reinjects canonical initial context at a model-compatible boundary. If the compaction request itself overflows, Codex progressively drops the oldest history item and retries while keeping recent items. [OpenAI compact implementation](https://github.com/openai/codex/blob/main/codex-rs/core/src/compact.rs)
- Compaction is a first-class thread item with lifecycle events and analytics. The app-server exposes manual `thread/compact/start`, while clients can also observe automatic compaction. [OpenAI app-server protocol](https://github.com/openai/codex/blob/main/codex-rs/app-server/README.md)
- OpenAI explicitly warns that repeated compactions reduce accuracy and recommends starting a targeted new thread when possible. Codex preserves coding continuity well, but its compaction summary is still lossy and is not a durable business-task authority. [OpenAI compact implementation](https://github.com/openai/codex/blob/main/codex-rs/core/src/compact.rs)

## Assessment

Codex remains stronger in provider-side remote compaction, canonical initial-context reinjection, mature thread protocol lifecycle, and aggregate analytics. Thruvera now closes the major local-mechanics gaps around model-aware budgets, oversized-input bounding, lifecycle events, and post-summary preservation checks.

Thruvera is stronger at separating conversation history from durable work authority. A Task, Checkpoint, Effect state, Verification state, and Memory record survive independently of the summary; the Task Preservation Envelope helps the model resume without treating chat prose as authoritative.

Thruvera now emits a deterministic multilingual semantic-coverage proxy for durable continuation anchors alongside exact identity coverage. Its remaining highest-value steps are before/after token and latency aggregates, repeated-compaction soak tests over Task/Checkpoint invariants, and—only if measurements justify the cost—an optional provider-backed quality evaluator. It should retain Pi as the sole session/compaction implementation and deepen the Thruvera preservation and observability layer rather than create another compactor.
